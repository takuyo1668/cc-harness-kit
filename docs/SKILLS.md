# SKILLS — 収録スキル・コマンドの使い方

> **スキルとは**: 特定の作業のための手順・チェックリストをまとめたもの。Claude が状況に応じて自動で
> 呼び出すか、`/スキル名` で明示的に起動する。`~/.claude/skills/<名前>/SKILL.md` が本体。

---

## 収録スキル（自作・汎用）

### logical-thinking — 思考フレームワーク索引
思考タスク（分析・課題分解・意思決定・優先順位付け・戦略・アイデア発想・振り返り・構成）の**着手前**に、
85種のフレームワークの索引から最適な型を最小トークンで選ぶルーター。
- 使いどき: 「整理して」「分析して」「なぜ」「どれを選ぶ」「戦略」「構成を考えて」等
- 効果: いきなり書き始めるより、型を1つ選んでから考える方が精度が上がる
- 起動: 自動、または該当タスクの前に自分で参照

### verification-before-completion — 完了報告前ゲート
実装・修正・設定変更を「完了しました」と言う直前に、証拠なき完了報告（推測完了報告）を封じる自己照合表。
- 使いどき: 「完了」「修正しました」「動くはず」と言おうとしたとき、PR/デプロイ前
- 効果: 「テストを実行していないのに完了と言う」を機械的に防ぐ。**このキットで最も効くスキルの一つ**

### design-review — UI/UX 多角レビュー（学習型）
実装コード（HTML/CSS/JSX）を ui-specialist / product-designer / tech-researcher の3エージェント並列で
レビューし、UI品質・UX・アクセシビリティを多角評価する。
- 起動: `/design-review <パス>`（`--quick` で軽量、`--no-feedback` で学習スキップ）
- 学習: レビュー後のフィードバックを `preferences.md` / `patterns.md` に蒸留し、使うほど「あなたの好み」を
  学習して指摘精度が上がる。**配布時は学習内容をリセット済み**なので、あなたの環境で白紙から育つ。

### ui-design-quality — 大量画面のデザイン品質維持
UIの品質を3層（L1/L2/L3）で定義し、Wave方式で大量画面を品質を保ちながら制作する。
- 使いどき: Figmaデザイン制作、画面設計、デザインシステム構築

### oss-import — OSS取り込み判断ゲート
GitHub上のOSS（スキル集・プラグイン・サブエージェント・dotfiles）を自分の `~/.claude` へ**安全に**
取り込むかを判断するワークフロー。丸ごと導入・思想だけ自前移植・部分コピーのどれを選ぶかのゲート。
- 使いどき: 「このリポジトリを環境に活かしたい」「このプラグイン入れていい?」

### setup-kit-share — 配布キット化
自分の環境（フォルダ構成・スキル・自動化）を、他人がまっさらなPCで再現できるキットに変換する。
**このキット自体がこのスキルで作られた。** 横展開・社内配布・新メンバー用セットアップに使う。

### slack — Slack共有文の選択式生成
保存した記事・ツール・ナレッジを、要素ごとに3〜5案を提示する選択式ワークフローでSlack共有文に変換する。

### japanese-writing — 日本語ライティング規範（違和感除去＋認知リズム）
日本語の文章から「AIっぽさ」（LLM口調・翻訳調・機械的な違和感）を消し、読み進める推進力を持つ文章に
仕上げる2層の執筆・推敲規範。層1で文レベルの減点（「重要なのは〜」「〜することができます」「いかがでしたか」等）を
機械的に除去し、層2で緩急（文の拍・未回収の緊張）を設計する。
- 使いどき: 記事・note・レポート・提案書本文・READMEを書くとき、「リライトして」「自然な日本語に」「AIっぽさを消して」
- 効果: 判定軸は「その文が更新するのは状況か文書か」の一つだけ。曖昧な「良い文章を」ではなく機械的に点検できる
- 起動: 自動（執筆・推敲の依頼で発動）。LP文言・広告コピーは対象外（copywriting系の領域）
- 出典: k16shikano（ラムダノート・鹿野桂一郎）の公開規範（Unlicense）を一般文書向けに再構成＋AI日本語特有の型を追加

## 収録コマンド

| コマンド | 用途 |
|---|---|
| `/pm` | プロジェクト管理。22名の専門エージェントから動的にチーム編成し、意図深掘り→計画→並列実行→品質チェック→統合→振り返りで進める |
| `/save` | 会話の現状（完了/未完了タスク・意思決定）を memory に保存し次回再開可能に |
| `/weekly-review` | 週次の活動とフィードバックを集計し、ルールの死荷重を見直す |
| `/ship-check` | デプロイ前の最終チェック（テスト/lint/型/秘密情報/破壊的変更） |
| `/market-research` | 市場・競合・ユーザー調査を並列リサーチ→ファクトチェック→多角レビューで実施 |
| `/health-check` | Mac環境診断（メモリ・ディスク・プロセス）。ファイル削除は提案のみ |

## このキットに「含めていない」スキルについて

Anthropic 公式プラグインのスキルは、各自プラグインから入手できるため**同梱していません**。代表例:
- マーケティング系: `copywriting` `copy-editing` `page-cro` `paid-ads` `ad-creative` `analytics-tracking`
  `ab-test-setup` `pricing-strategy` `social-content` `customer-research` `churn-prevention` など
- 制作・汎用系: `skill-creator` `systematic-debugging` `theme-factory` `frontend-slides` `dataviz`
  `artifact-design` `grill-me` など

これらは Claude Code のプラグイン機能（`/plugin` 系）や公式マーケットプレイスから導入してください。

また、配布者の会社・個人業務に固有のスキル（社内エンティティ辞書・特定事業のリサーチ自動化・
個人のノート運用連携など）は、汎用性が無いため除外しています。必要なら `skill-creator` で
自分の業務に合わせて作るのがおすすめです。
