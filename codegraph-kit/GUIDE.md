# codegraph-kit — コードのナレッジグラフ化で「変更の影響範囲」を一瞬で見えるようにする

Claude Code に **CodeGraph**（コードの構造を事前に索引化するオープンソースツール）を接続し、
「この関数を変えたらどこが壊れるか」「この機能はどこと繋がっているか」を1コマンドで答えられるようにするキットです。

> **CodeGraph とは**: プロジェクト内の全関数・呼び出し関係・依存関係を SQLite データベースに索引化（ナレッジグラフ化）し、
> AI エージェントが grep やファイルの逐次読みをせずに「関連コード＋呼び出し経路＋影響範囲」を1回の問い合わせで得られるようにする
> MIT ライセンスの OSS。100% ローカル動作で、コードが外部に送信されることはありません。
> 出典: https://github.com/colbymchenry/codegraph

---

## これで何が変わるか

1. **変更前に「壊れる場所」がわかる** — 例: 実際の社内プロジェクトで `getRedirectUri`（Google認証の関数）の
   影響範囲を聞くと「Drive・Calendar・Gmail 連携の計7関数に波及する」と数秒で返る。手作業で追うと数十分の調査です。
2. **Claude の探索が速く・安くなる** — 公式ベンチマーク実測でツール呼び出し58%減・ファイル読み込みほぼゼロ。
   「どう動いているか」系の質問が1コールで済みます。
3. **索引の鮮度管理が不要** — ファイル監視で編集のたびに自動更新。新しく作った機能も自動でグラフに連結されます。

## セットアップ（あなたがやること）

1. このフォルダの `PROMPT.md` を開き、中の指示文をコピーする
2. 自分の Mac で Claude Code を起動し、貼り付けて送信する
3. あとは Claude が質問しながら進めます（所要 5〜10分。インデックス構築はプロジェクト規模により数十秒〜数分）

## 日常の使い方（覚えるのは3つだけ）

セットアップ後、対象プロジェクトで Claude に普通に質問すれば自動で使われます。手動で使う場合:

| コマンド | 何がわかるか |
|---|---|
| `codegraph explore "<質問や関数名>"` | 関連コード＋呼び出し経路＋影響範囲を一括表示 |
| `codegraph impact <関数名>` | その関数を変更したとき影響を受ける場所の一覧 |
| `codegraph callers <関数名>` | その関数を呼んでいる場所の一覧 |

## FAQ

- **費用は？** — 無料（MIT ライセンスの OSS）。API キーも不要。
- **コードは外部に送られる？** — 送られません。索引はプロジェクト内の `.codegraph/` フォルダ（ローカル SQLite）に置かれ、
  このキットでは匿名利用統計（テレメトリ）も無効化します。
- **どんなプロジェクトに向く？** — 目安はソースファイル50以上。数ファイルの小さなスクリプト集には効果がありません。
- **索引の更新は？** — 自動です。Claude Code セッション中はファイル監視、セッション外の変更も次回接続時に差分同期されます。
- **限界は？** — コードの索引なので、DB スキーマ・環境変数・外部サービス設定が原因の不具合は検出できません。
  また「変更後に壊れていないことの証明」はテストの仕事です（このツールは「どこを確認すべきか」を教えるもの）。
- **やめたいときは？** — プロジェクトの索引削除は `codegraph uninit <パス>`、本体削除は `npm rm -g @colbymchenry/codegraph`。
  設定ファイル（`.mcp.json` など3点）の削除方法は SETUP.md 末尾に記載。

## 困ったら

- 「CodeGraph not initialized」→ そのプロジェクトで `codegraph init` を実行
- シンボルが見つからない → 2秒ほど待つか `codegraph sync`。対象言語・除外設定（.gitignore）も確認
- それでも解決しなければ、Claude Code に「codegraph の調子が悪い。`codegraph status` から診断して」と頼む
