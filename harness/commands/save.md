会話の現状を保存して、次回の会話で即座に再開できるようにしてください。

## 保存条件チェック
以下のいずれかを満たさない場合は「保存不要です」と伝えてスキップ:
- 実装タスクや設計判断を行った
- 未完了のタスクが残っている
- 複数ターンのやり取りで意思決定が発生した

## 手順

### 1. 関連プロジェクトメモリの更新
- 今回の作業に関連する `project_*.md` メモリを特定する
- ステータス、ネクストアクション、リンク（URL等）を最新状態に更新する
- 該当メモリがなければ新規作成する

### 2. memory に会話ログを保存（プライマリ）
`~/.claude/projects/<slug>/memory/conversation_log_YYYYMMDD.md` に保存（slug は作業ディレクトリから自動生成される）:
- ステータス（完了 / 途中 / 回答待ち）
- 完了済みタスクの概要
- 未完了タスク・次のアクション（プロジェクトメモリへの参照でOK）
- 重要な意思決定とその理由
- 関連ファイルのパス
- 同日の既存ログがあれば内容を更新（上書き）

### 3. （任意）外部ノートアプリにも保存
Obsidian などのノートアプリを併用している場合は、閲覧用に同じ内容を保存する。保存先パスは自分の環境に合わせて設定する（未使用ならこの手順はスキップ）。テンプレート例:
```markdown
---
title: Claude Code ログ YYYY-MM-DD
date: YYYY-MM-DD
tags: [claude-log]
type: daily
status: active（完了なら done）
related: []
---
## ステータス: 完了 / 途中 / 回答待ち
## 完了タスク
## 未完了タスク・次のアクション
## 重要な意思決定
## メタ認知ジャーナル
- 判断:
- 根拠:
- 次回の省略:
## 関連ファイル
```

### 4. MEMORY.md の更新
- 新規メモリを追加した場合はインデックスに追記
- 既存項目の説明文が古くなっていれば更新
- conversation_log が15件を超えたら、完了済みかつ2週間以上前のログを「インデックス行とファイルをセットで」 ~/.claude/archive/ へ移動を提案（移動はユーザー承認後）

### 5. 完了報告
保存した内容のサマリーと、再開時に使うキーワード（例:「X自動化の続き」）を伝える。
