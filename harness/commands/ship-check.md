---
description: デプロイ前の最終チェック。テスト/lint/型/秘密情報/破壊的変更を一括確認
---

以下を順に実施し、PASS/FAIL一覧で報告してください。

## チェックリスト

- [ ] **テストスイート**: 全PASS（`npm test` / `pytest` 等）
- [ ] **lint**: 0エラー（`npm run lint` / `ruff check` 等）
- [ ] **型チェック**: 0エラー（`tsc --noEmit` / `mypy` 等）
- [ ] **ビルド**: 0エラー（`npm run build` 等）
- [ ] **git status**: コミット済み（未追跡ファイルなし or 意図的）
- [ ] **.env / 秘密情報ファイル**: 未追跡確認（`.gitignore` 記載確認）
- [ ] **APIキー / トークン**: コード・コメント・ログに平文ハードコードなし
- [ ] **破壊的変更**: DBマイグレーション・API破壊的変更を明示
- [ ] **ロールバック手順**: 提示済み
- [ ] **環境変数**: 本番環境への追加・変更が必要なら明示

## 本番デプロイ先確認

デプロイ先が本番環境（Railway production / GAS本番 / Vercel prod 等）の場合:
- 「本番環境にデプロイします。対象: [環境名]。よいですか？」と確認
- ユーザーの明示承認（yes / 実行 / OK）を待つ

## DB マイグレーションを含む場合

- ドライラン結果を提示
- 影響行数を明示
- カラム削除・テーブル削除を含む場合「データが失われます」と警告
- バックアップ済みか確認

## 言い訳セルフチェック（推測完了報告の封じ）

上記が全PASSでも、報告前に `verification-before-completion` スキルの言い訳封じ表で自己照合する。
「動くはず」「テストは後で」「たぶん通る」等の言い訳に該当していないか確認し、
該当する行があればその「とるべき行動」を実施し**証拠（実行ログ・スクショ・テスト出力）を提示してから** PASS とする。

## 報告フォーマット

```
【Ship Check 結果】
✅ テスト: PASS
✅ lint: 0エラー
✅ 型チェック: 0エラー
✅ ビルド: 0エラー
⚠️ git status: 未コミットの変更3ファイル
✅ 秘密情報: 検出なし
❌ DBマイグレーション: ロールバック手順未提示

→ デプロイ推奨: NO（❌ 1件あり）
→ 対応すべき項目: [リスト]
```

## 重要

**1つでもFAILがあればデプロイを推奨しない**。修正完了後に再実行する。
