今週の活動を振り返り、ハーネスを点検して来週の改善目標を設定してください。

## 実行手順

### 1. 今週の活動分析
- memory ディレクトリ（`~/.claude/projects/<slug>/memory/`。slug は作業ディレクトリから自動生成される）内の会話ログ（直近7日分）を読み込む
- 今週実行したタスクを一覧化

### 2. フィードバック集計
- feedback メモリを集計し、今週のパターンを分析
- 根本原因カテゴリ別の件数: 前提ミス / 過剰設計 / 検証不足 / コンテキスト不足
- 同一カテゴリが3回以上 → ワークフロールール強化を提案

### 3. ハーネス死荷重レビュー（"What can I stop doing?"）— 月1回（第1週）は必須
対象: ~/CLAUDE.md / ~/.claude/rules/*.md / ~/.claude/commands/ / ~/.claude/agents/ / ~/.claude/skills/ / memory
- 各ルールを判定:
  - **まだ必要**: 今週このルールが品質を守った実績がある → 維持
  - **過剰かも**: ボトルネックになった、または発動しなかった → 緩和を提案
  - **不要**: Claudeの現在の能力で自然にできている → 削除を提案
- 過去1ヶ月使っていない commands / skills / agents を死蔵候補として報告
- MEMORY.md の項目数を確認: 80項目 or 15KB 超過なら、完了済みかつ2週間以上前の conversation_log を「インデックス行とファイルをセットで」 ~/.claude/archive/ へ移動を提案
- ルール内の日付・期限・プロジェクト名が現実と一致しているか確認（鮮度チェック）
- prompt-review ルーブリック（~/.claude/skills/prompt-review/rubric.md）の鮮度確認: Anthropicドキュメント（冒頭の出典URL）の更新有無をチェックし、差分があれば反映
- 変更提案はユーザー承認後に実施。削除したルールは理由を feedback メモリに記録

### 4. 週次レポートの出力

```
# Weekly Review — YYYY-MM-DD

## 今週の活動
- [タスク一覧と成果]

## 失敗パターン分析
- [カテゴリ別集計と傾向]

## ハーネス死荷重レビュー
| 対象 | 判定 | 理由 |
|---|---|---|
| [ルール/コマンド名] | 維持/緩和/削除 | [今週の実績ベース] |

## 来週の改善目標
1. **フォーカス**: [1つに絞る]
2. **具体的アクション**: [実践で取り組むこと]
3. **達成基準**: [何ができたらOKか]
```
