# raw/05 戦略の方向性・表現方法・目標設定

思考プロセスマップの「⑥ 目標・計画に落とす（＋戦略の方向性・表現）」。
★=コア（早見表で即使える／ここは深掘り版）。

---

### クロスSWOT★

- **一言**: SWOTの4要素を2×2で掛け合わせ「実行可能な戦略オプション」を4種類に変換する。
- **使う場面（トリガー）**: 状況整理（SWOT）は終わったが「で、何をするか」の打ち手が出ていない／新規事業・中期計画・競合台頭時の戦略見直し。SWOT止まりで資料が止まるのを防ぐ。
- **手順**:
  1. 先にSWOTを完成（S/W/O/T各3〜5項目）
  2. 2×2に SO・ST・WO・WT のラベル
  3. SO（強み×機会）=積極攻勢
  4. ST（強み×脅威）=差別化・防衛
  5. WO（弱み×機会）=改善・提携
  6. WT（弱み×脅威）=縮小・撤退・リスク遮断。全セル後、実現性×インパクトで上位2〜3戦略に絞る
- **なぜ効率的か**: SWOT止まりの「現状分析レポート」を戦略行動に直結させるブリッジ。4セルの型が抜けを防ぎ議論の発散をなくす。
- **落とし穴**: SWOT要素の粒度がバラバラだと組合せがチグハグ（事実ベースで具体的に）／WT戦略を弱気に見えると省略しがちだがリスクヘッジに必須。
- **相性**: [[SWOT分析]]（前段）、3C分析（SWOT前の情報収集）、アンゾフ（SO戦略の方向性）、[[KPIツリー]]（選定戦略の成果指標化）

### KPIツリー★

- **一言**: 最終目標（KGI）を四則演算で分解し「どの指標をどれだけ動かせば達成か」を木構造で可視化。
- **使う場面（トリガー）**: 「売上が足りない」「CVRが悪い」の原因特定と改善優先順位付け／SaaS・EC・広告など数値管理／チームで「何を追うか」の意思統一。
- **手順**:
  1. KGI（最終目標）を1つ決める（例 月次売上1,000万円）
  2. 四則演算で分解（売上=受注数×顧客単価）
  3. さらに分解（受注数=商談数×受注率…）
  4. 「計測可能か」「重複なく合算・積算でKGIに戻るか」をMECEチェック
  5. 各KPIに現状値・目標値を入れボトルネック（最大ギャップ）を特定
- **なぜ効率的か**: 「どこを改善すれば最大インパクトか」が一目で分かり、感覚でなく構造で優先順位を決められる。左=遅行指標／右=先行指標で施策を先読みできる。
- **落とし穴**: 定性指標を混在させ因果が崩れる／同じ指標を複数枝に重複計上（ダブルカウント）／KPIが多すぎ「全部追う」になる（上位2〜3層に絞る）。
- **相性**: AARRR（SaaSのKGI分解）、[[クロスSWOT]]（戦略のKPI化）、SMART（各KPIの目標値設定）、ガントチャート（改善施策のスケジュール化）

### プロダクト・ポートフォリオ・マネジメント（PPM / BCGマトリクス）

- 使う場面: 複数事業・商品への経営資源配分を決める。事業整理・新規投資・撤退基準の合意形成。
- 構成: 縦=市場成長率／横=相対的市場シェアの4象限。花形（高成長×高シェア・投資維持）／金のなる木（低成長×高シェア・収益最大化）／問題児（高成長×低シェア・選択投資）／負け犬（低成長×低シェア・撤退検討）。金のなる木のキャッシュを花形・問題児へ。

### アンゾフの成長マトリクス

- 使う場面: 次の成長方向を「製品×市場」の4択で選ぶ。中期戦略・新規事業・ピボット判断。
- 構成: 市場浸透（既存×既存・最低リスク）／製品開発（既存市場×新製品）／市場開発（新市場×既存製品）／多角化（新×新・最高リスク）。右下ほど高リスク。

### STP

- 使う場面: 「誰に・何を・どう伝えるか」の戦略軸を決める。LP・広告・新サービスの起点で4Pの前に通す。
- 構成: S=Segmentation（市場細分化）／T=Targeting（自社の強みが最大化するセグメント選定）／P=Positioning（競合との差別化軸定義）。基本はこの順、P→T→Sの逆順が有効な場合も。

### ポジショニングマップ

- 使う場面: 競合比較を視覚化し空いているポジション（ホワイトスペース）を見つける。STPのP段階・広告コンセプト・差別化。
- 構成: 2軸（価格×品質 等）のXY平面に競合と自社をプロット。軸は「顧客が購買判断に使う基準2つ」にすると意味が出る。恣意的な軸だと判断を誤る。

### ビジネスモデル・キャンバス（BMC）

- 使う場面: 新規事業のビジネスモデルをA3一枚で構造化。ピッチ前・設計合意・既存事業の棚卸し。
- 構成（9ブロック）: 顧客セグメント／価値提案／チャネル／顧客との関係／収益の流れ／主要リソース／主要活動／パートナー／コスト構造（オスターワルダー）。左=価値を生む仕組み、右=価値を届ける仕組み、中央=価値提案、下=お金の流れ。

### スキーム図

- 使う場面: 関係者・お金・情報・権利の流れを一枚で説明（契約関係・API連携・資金調達スキームを非専門家に）。
- 構成: 「誰が（箱）・何を（矢印ラベル）・誰に（箱）・流す（向き）」が基本。方向は左→右or上→下で統一。矢印の意味（資金/情報/契約/権限）を凡例で明示。

### AIDMA

- 使う場面: 購買プロセス設計。主にオフライン・マスメディア主体（高関与・検討期間が長い）商材。
- 構成: A=Attention(認知) / I=Interest(関心) / D=Desire(欲求) / M=Memory(記憶) / A=Action(行動)。リピート購買・想起率が重要なブランドで有効。

### AISAS

- 使う場面: ネット・SNS環境下の消費者行動設計。EC・SaaS・アプリのファネル、SEO・SNS広告の役割分担（電通, 2005）。
- 構成: A=Attention / I=Interest / S=Search(検索・比較) / A=Action / S=Share(共有)。AIDMAのD・MがSearch・Shareに置換。Shareが次のAttentionを生む循環が特徴。

### ガントチャート

- 使う場面: タスク・担当・期間・依存関係を時系列管理。開発スプリント・キャンペーン進行・複数人プロジェクトのスケジュール合意。
- 構成: 縦=タスク／横=時間軸／バー=担当と期間。マイルストーンを菱形で、依存関係を矢印で接続するとクリティカルパスが見える。

### 組織図

- 使う場面: 指揮命令系統・役割分担の可視化。採用時の役割説明・責任範囲の明確化・M&A後の統合設計。
- 構成: 箱（役職・人名）と線（報告ライン）。ツリー型／マトリクス型／フラット型。ライン部門とスタッフ部門を線種で区別。

### ロードマップ

- 使う場面: 中長期の方向性・優先順位・マイルストーンを全員で共有。製品開発・事業戦略の合意・投資家説明。ガントより粗く「何をどの順序で」が主目的。
- 構成: 時間軸（横）×テーマ・フェーズ（縦）に機能・目標・マイルストーンを配置。Now/Next/Laterの3ゾーンや四半期区切りが実用的。

### AARRR（海賊指標 / Pirate Metrics）

- 使う場面: SaaS・アプリ・ECの成長ファネルを段階別に管理。グロースのKPI設計・優先順位・ボトルネック特定（Dave McClure, 2007）。
- 構成: A=Acquisition(流入) / A=Activation(初期体験) / R=Retention(継続) / R=Revenue(収益化) / R=Referral(紹介)。最も離脱する段階に集中打開。Retentionが弱いと他を強化しても穴の空いたバケツ。

### SMART

- 使う場面: 目標が曖昧で「達成したか判断できない」状態を防ぐ。OKRのKR策定・KPIへの数値割当・行動計画。
- 構成: S=Specific(具体的) / M=Measurable(測定可能) / A=Achievable(達成可能) / R=Relevant(関連性) / T=Time-bound(期限)。5要素すべてで「機能する目標」に。A=Agreed-upon、R=Result-orientedと解釈する流派も。
