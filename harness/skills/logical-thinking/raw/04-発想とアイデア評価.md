# raw/04 アイデア発想・形にする・評価と選択

思考プロセスマップの「④ アイデアを発想する／⑤ 評価・選択する」。
★=コア（早見表で即使える／ここは深掘り版）。

---

### ペイオフマトリクス（Payoff / Impact-Effort Matrix）★

- **一言**: アイデアを「効果（インパクト）」×「実現容易性（労力）」の4象限に配置し、何を先にやり何をやらないかを可視化する優先順位決定ツール。
- **使う場面（トリガー）**: ブレスト後にアイデアが10件以上出て選別が必要／限られた時間・予算・人員で着手順を決める／チームで施策の優先度認識がずれている。
- **手順**:
  1. 縦=効果（高→低）、横=実現容易性（高→低）の2×2を描く
  2. 各アイデアを各軸3〜5点でスコアリング
  3. マトリクス上に配置
  4. 第1象限（効果高×容易）=即実行、第2（効果高×困難）=要計画、第3（効果低×容易）=余裕があれば、第4（効果低×困難）=見送り
  5. 第1象限から優先度を確定
- **なぜ効率的か**: 主観的な「やりたい順」でなく効果とコストを同時に可視化し、リソース配分の根拠を一枚で説明。会議の収束が速い。
- **落とし穴**: 「実現容易性」のスコアが人によりブレる（工数・予算の定義を先に揃える）／効果が直感だと第1象限に人気施策が集中し選別の意味が消える／環境変化で再評価が必要。
- **相性**: ブレインライティング・マンダラート（量産フェーズの出力を絞る）、プロコン表（第2象限「要計画」の詳細評価）、意思決定マトリクス（軸を増やした精緻版）

### ブレインライティング（635法）

- 使う場面: 声の大きい人が場を支配するブレスト、内向的メンバーが多いチームのアイデア出し。
- 手順: 6名が各自シートにアイデア3件記入→隣に回す→他者の案を見つつ3件追記→5回転で計108件（635法）。非同期はスライド/付箋ツールで代替可。

### マンダラート（Mandal-Art）

- 使う場面: 中心テーマから派生要素を構造的に展開したいとき（目標設定・アイデア整理・業務棚卸）。
- 手順: 3×3の中央にテーマ→周囲8マスにサブテーマ→各サブテーマを別の3×3中央に移して展開→最終的に9×9=81マス。

### 形態分析法（Morphological Analysis / Zwicky Box）

- 使う場面: 商品開発・サービス設計で「要素の組合せパターン」を網羅探索したいとき（ツビッキー考案）。
- 手順: 問題をパラメータに分解→各パラメータの選択肢を行に並べた形態分析表→各行から1つずつ選び組合せを列挙→有望な組合せを評価・選定。

### シナリオグラフ（シナリオプランニング図）

- 使う場面: 将来不確実性が高い意思決定（新規事業・中期計画）で複数の未来を描き最悪・最良・最確を比較。
- 手順: 影響度と不確実性が高い変数を2軸に→4象限に4シナリオ名→各象限に具体事象・数値・前提→シナリオごとに取るべき戦略を対応。

### オズボーンのチェックリスト

- 使う場面: 既存製品・プロセスの改善が行き詰まったとき、9視点の問いで思考を解凍。
- 手順（9項目）: 転用／応用（借用）／変更／拡大／縮小／代用／置換・再配列／逆転／結合。各問いを順に当てる。

### SCAMPER

- 使う場面: 既存アイデアに7種の強制変換問いを当て改良・派生案を体系生成（オズボーンをエバーリーが再整理, 1971）。
- 手順（7問）: S=Substitute(代替) / C=Combine(結合) / A=Adapt(適応) / M=Modify・Magnify(変形・誇張) / P=Put to another use(転用) / E=Eliminate(削除) / R=Reverse・Rearrange(逆転・再配列)。

### アイデアシート

- 使う場面: 発想段階のアイデアを後で評価・共有できるよう構造化記録。
- 手順: 1枚1アイデアで「タイトル・課題・解決策・ターゲット・期待効果・次のアクション」を埋める。全員同一フォーマットで比較可能性を確保。

### ストーリーボード

- 使う場面: UX・サービスフロー・広告シナリオを時系列コマ割りで視覚化し共通認識を作る。
- 手順: 行動を時系列に並べた6〜8コマ→各コマに場面・行動・感情・接点を記入→遷移の矛盾を確認→抜けたシーンを補う。

### プロコン表（Pros/Cons）

- 使う場面: 2〜3の選択肢から1つを選ぶ意思決定で賛否の根拠を整理し判断を透明化。
- 手順: 選択肢ごとにPro（メリット）とCon（デメリット・リスク）を列挙→重要度で重み付け（1〜3点）→合計で比較、または致命的Conを「否決条件」にして絞る。

### SUCCESs（Made to Stick 6原則）

- 使う場面: アイデア・提案が「伝わらない・記憶されない・行動につながらない」とき、メッセージの診断・強化（ハース兄弟『アイデアのちから』2007）。
- 手順（6原則でチェック）: S=Simple(単純・核心1点) / U=Unexpected(意外性) / C=Concrete(具体) / C=Credible(信頼・裏付け) / E=Emotional(感情) / S=Stories(物語)。欠けた原則を特定し書き直す。
