# 思考フレームワーク・ルーター

85種のロジカルシンキングFWを「思考の段階」で引き出すための索引。
**目的: 思考の順番を最適化し、最小トークンで最大精度のアウトプットを出す。**

## 使い方（このプロトコルを守る）

1. **思考の段階を特定**する（下の「思考プロセスマップ」のどこにいるか）
2. **判断表で第一候補FWを選ぶ**（トリガー＝今の状況シグナルで引く）
3. **下の「コアFW早見表」で足りればそのまま思考開始**。手順・落とし穴・組合せの深掘りが要るときだけ、該当の `raw/*.md` を**1ファイルだけ**Readする
4. 複数段階にまたがる場合はFWを**順に連結**する（例: なぜなぜ→ロジックツリー→意思決定マトリクス）

> **トークン節約の鉄則**: raw全体やSKILL全文を毎回読まない。索引（この表）で当たりをつけ、必要な1ファイルの該当節だけ開く。コア18個は早見表で即使える。

## 思考プロセスマップ（迷ったらこの順番）

```
① 問題を発見する   → ② 整理・構造化する → ③ 分析する（環境/顧客/競合）
   （ギャップ/真因）     （MECE/ツリー）        （3C/SWOT/PEST 等）
        ↓                                              ↓
⑧ 伝える          ← ⑦ 振り返る・改善する ← ⑥ 目標・計画に落とす
   （PREP/結論先行）    （KPT/PDCA）           （KPIツリー/SMART/ロードマップ）
        ↑                                              ↑
   （説明・説得）      ⑤ 評価・選択する  ←  ④ アイデアを発想する
                        （ペイオフ/意思決定M）   （SCAMPER/マンダラート 等）
```

## コアFW早見表（★18個・raw無しで即使える）

これだけは手順を内在化しておく。深掘りが要る時のみ各rawを開く。

| FW | 一言の型 | 使いどころ |
|---|---|---|
| So What?/Why So? | 「つまり何が言える？」「なぜそう言える？」で往復 | 論理の飛躍チェック |
| 空・雨・傘 | 事実(空)→解釈(雨)→打ち手(傘) | 提案の最小構成 |
| 帰納法 | 複数事実→共通点→一般化 | データから結論 |
| 演繹法 | 一般則＋個別事実→結論 | ルールから判断 |
| As is/To be | 理想−現状＝課題 | 問題定義 |
| なぜなぜ分析 | 「なぜ」を5回、真因まで | 再発防止の根治 |
| MECE | モレなく・ダブりなく | あらゆる分解の前提 |
| ロジックツリー | What/Why/How で階層分解 | 問題分解・原因探索・打ち手出し |
| ピラミッドストラクチャー | 結論→根拠3つ→各根拠の事実 | 説明・文書構成 |
| 緊急度/重要度マトリクス | 重要×緊急の4象限 | タスク優先度 |
| 意思決定マトリクス | 選択肢×評価軸×重み付け点数 | 客観的選択 |
| 3C分析 | Customer/Company/Competitor | 状況把握の起点 |
| SWOT分析 | 強み弱み×機会脅威 | 内外要因の棚卸し |
| クロスSWOT | SWOTを掛け合わせ4戦略 | SWOT→具体戦略 |
| ペイオフマトリクス | 効果×実現性の4象限 | 施策の優先度 |
| KPIツリー | KGI→KPI→打ち手に分解 | 目標を行動に変換 |
| PDCA | 計画→実行→評価→改善 | 継続的改善 |
| KPT | Keep/Problem/Try | 振り返り |
| PREP | Point→Reason→Example→Point | 結論先行で伝える |

## 連結パターン（よく効く順番）

- **問題解決の王道**: As is/To be →（なぜなぜ／Whyツリー）→ MECEで打ち手出し → ペイオフ／意思決定マトリクスで選択 → KPIツリーで実行管理 → KPTで振り返り
- **戦略立案**: 3C／PEST → SWOT → クロスSWOT → STP → 4P
- **提案資料**: 空・雨・傘 or ピラミッド で骨子 → PREP で各スライド
- **アイデア出し→実行**: SCAMPER／マンダラートで発散 → ペイオフで収束 → ストーリーボードで具体化

## メンテナンス

- 各FWの詳細手順・落とし穴・組合せは `raw/01〜06` が正本。コアは深掘り、その他は圧縮。
- FWを追加・修正したら、この判断表とコア早見表のFW名も同期する（正本はこのSKILL.md）。
