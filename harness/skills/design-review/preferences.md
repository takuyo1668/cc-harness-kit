# ユーザーの美的好みプロファイル

design-review スキルが毎回参照する「ユーザー固有のデザイン好み」を蓄積するファイル。
レビュー実行 → ユーザーFB収集 → このファイルに蒸留 のサイクルで育てる。

**重要**: このファイルの内容はレビュー時に各エージェントへ「ユーザーの好みプロファイル」としてプロンプト注入される。
**抽象的な美辞麗句は書かない**。具体的・検証可能な好み傾向のみ書く。

> **配布時リセット済み**。ここから先はあなた自身のレビュー履歴で育つ。下のコメント例はフォーマットの見本。
