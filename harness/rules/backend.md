# バックエンド作業時のルール

API・サーバーサイド・DB処理を含む作業では以下を遵守する。

## 外部API書き込み
- POST / PUT / PATCH / DELETE はコメントに「外部書き込みあり」明記
- テスト実行時はサンドボックス・開発用リソースに向ける
- 認証情報をログに出さない。デバッグログは変数名のみ（例: `console.log("TOKEN exists:", !!process.env.TOKEN)`）

## エラーハンドリング
- 構造化ログで出力（メッセージ・スタックトレース・コンテキスト）
- サイレント失敗禁止。業務代替機能はエラーを必ずUIで人間に通知
- 外部APIはレート制限・タイムアウト・リトライ戦略を明示

## DB アクセス
- N+1 を避ける
- マイグレーション手順（ドライラン・影響行数・データ損失警告・ロールバック）は infrastructure.md「DBマイグレーション」が正本
- スキーマ変更・大量UPDATE/DELETE前に「バックアップ済みか」確認

## 認証・セキュリティ
- 秘密情報は環境変数のみ。コードにハードコード禁止
- セッショントークン・JWT・APIキーをCLAUDE.mdやコメントに書かない
- 入力検証は境界（HTTPハンドラ・外部API応答）で実施

## GAS / Slack Bot 等
- デプロイ前にスクリプトのトリガー設定への影響を確認
- 自動化トリガー（GAS time-based / Cron）に送信ロジックを追加する場合は承認必須
- clasp push は本番スクリプトIDの場合特に承認必須
