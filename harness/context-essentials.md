## 本質的原則（Boris Cherny / Claude Codeリード運用哲学）

1. **Plan modeはデフォルト**: 単純修正以外は必ずPlanから。良いPlanが成果の大半を決める
2. **検証ループが品質を2-3倍にする**: 検証手段を必ず与える。検証できないとClaudeは推測する
3. **知識はPRに埋め込む**: ミスしたら即CLAUDE.md/feedbackメモリに追記
4. **繰り返し作業はslash command化**: 2回手でやったらコマンド化
5. **過度にカスタマイズしない**: 少ない・濃いルールが最強
6. **思考タスクはFWを選んでから**: 分析・課題分解・意思決定・戦略・発想・振り返り・構成などは、いきなり書かず `logical-thinking` スキルの索引で思考フレームワークを選んでから思考する

## 重要な不変原則（コンパクション後も保持）

- 手順は最初にまとめて提示。小出し禁止
- 不可逆操作は事前承認必須（rm -rf / git push --force / DROP TABLE / git reset --hard 等）
- テストファイル変更禁止（コードを直す。テスト変更が必要ならユーザー承認）
- 完了条件は検証可能な形のみ受け付ける（「良い」「適切」「きれい」等の曖昧語禁止）
- 5ファイル以上の変更は事前申告
- 同一エラー修正は3回まで→停止してエスカレート
- 秘密情報（APIキー・トークン）を出力・ログ・コメントに平文で書かない
- `git add .` / `git add -A` 禁止。個別ファイル指定
